# Firebase POI Experience Schema

## `pois/{poiId}`

Existing POI documents now include search fields generated by the backend when `upsertPoi` is called:

```js
{
  poiId: "gg_123",
  name: "Cafe Bien Xanh",
  normalizedName: "cafe bien xanh",
  normalizedAddress: "son tra da nang",
  category: "Cafe",
  location: { lat: 16.07, lng: 108.24, district: "Son Tra", address: "..." },
  searchKeywords: ["ca", "caf", "cafe", "bi", "bie", "bien", "xanh", "son", "tra"],
  rating: 4.5,
  ratingSum: 45,
  reviewCount: 10,
  timesAddedToItinerary: 24,
  timesVisited: 13,
  timesRouted: 31,
  status: "active",
  updatedAt: Timestamp
}
```

`searchKeywords` stores normalized tokens and prefixes for client autocomplete with:

```js
where("status", "==", "active")
where("searchKeywords", "array-contains", firstToken)
limit(15)
```

## `reviews/{reviewId}`

Created directly from the frontend after optional Firebase Storage uploads:

```js
{
  poiId: "gg_123",
  poiName: "Cafe Bien Xanh",
  userId: "firebase_uid",
  userName: "Nguyen An",
  userPhotoURL: "https://...",
  rating: 5,
  comment: "Không gian yên tĩnh, hợp làm việc.",
  visitPurpose: "work_study", // work_study | social | date | solo
  visitMood: "relaxed",       // relaxed | energetic | tired
  context: {
    dayOfWeek: "thursday",
    timeOfDay: "afternoon",   // morning | afternoon | evening | night
    weather: {
      source: "open-meteo",
      temperature: 31,
      precipitation: 0,
      weatherCode: 2,
      description: ""
    }
  },
  imageUrls: ["https://firebasestorage.googleapis.com/..."],
  imageCount: 1,
  status: "published",
  visibility: "public",
  createdAt: serverTimestamp(),
  updatedAt: serverTimestamp()
}
```

The UI subscribes by `poiId` and sorts client-side to avoid requiring a composite index for `where + orderBy`.

## `user_analytics/{eventId}`

Append-only behavioral telemetry written in the background from the HTML5 Geolocation loop.

POI dwell event:

```js
{
  eventType: "poi_visit",
  userId: "firebase_uid",
  poiId: "gg_123",
  poiName: "Cafe Bien Xanh",
  enterAt: Timestamp,
  exitAt: Timestamp,
  dwellMinutes: 28.4,
  distanceKm: 0.08,
  durationMinutes: 28.4,
  avgSpeedKmh: 0.17,
  inferredTransport: "walking",
  gpsAccuracyAvg: 18.5,
  sampleCount: 23,
  context: { dayOfWeek: "thursday", timeOfDay: "afternoon", weather: null },
  createdAt: serverTimestamp()
}
```

Route segment event between consecutive POIs:

```js
{
  eventType: "route_segment",
  userId: "firebase_uid",
  fromPoiId: "poi_1",
  fromPoiName: "Cafe A",
  toPoiId: "poi_2",
  toPoiName: "Restaurant B",
  startAt: Timestamp,
  endAt: Timestamp,
  durationMinutes: 12.2,
  distanceKm: 2.1,
  avgSpeedKmh: 10.3,
  inferredTransport: "cycling", // walking | cycling | motorbike_or_car | car
  gpsAccuracyAvg: 22.1,
  sampleCount: 18,
  context: { dayOfWeek: "thursday", timeOfDay: "evening", weather: null },
  createdAt: serverTimestamp()
}
```

## `_online_learning_events/{source_eventId}`

Internal idempotency marker written by Cloud Functions so retried Firestore triggers do not apply the same learning event twice.

```js
{
  userId: "firebase_uid",
  source: "review", // review | user_analytics
  eventId: "firestore_doc_id",
  processedAt: serverTimestamp()
}
```

Backend endpoints:

```http
POST /api/user-preferences/rebuild
POST /api/recommend-pois
```

`/api/recommend-pois` input:

```js
{
  userId: "firebase_uid",
  currentLocation: { lat: 16.05, lng: 108.20 },
  currentContext: {
    timestamp: "2026-06-11T09:30:00.000Z",
    dayOfWeek: "thursday",
    hour: 9,
    mood: "relaxed",
    visitPurpose: "work_study"
  },
  limit: 6
}
```

The recommender scores candidates with:

- active/open POI filtering hook
- global popularity: `timesVisited`, `timesAddedToItinerary`, `timesRouted`, ratings/reviews
- personalization: category, dwell, purpose, mood, and time-of-day affinities
- spatial distance from current GPS location using Haversine

## Atomic POI counters

The frontend uses Firestore atomic `increment()`:

```js
updateDoc(doc(db, "pois", poiId), {
  timesAddedToItinerary: increment(1),
  updatedAt: serverTimestamp()
})
```

Counters currently tracked:

- `timesAddedToItinerary`: incremented when a user adds a POI to an itinerary.
- `timesVisited`: incremented when geofencing records a successful dwell session.
- `timesRouted`: incremented when a user asks for navigation to a POI.
- `reviewCount` and `ratingSum`: incremented when a review is submitted.

## Firebase Storage

Review images are uploaded to:

```text
reviews/{poiId}/{userId}/{timestamp}-{index}-{filename}
```

The returned download URLs are stored in `reviews.imageUrls`.

## Suggested Rules

```js
match /pois/{poiId} {
  allow read: if true;
  allow write: if request.auth != null;
}

match /reviews/{reviewId} {
  allow read: if true;
  allow create: if request.auth != null
    && request.resource.data.userId == request.auth.uid
    && request.resource.data.rating >= 1
    && request.resource.data.rating <= 5;
  allow update, delete: if request.auth != null
    && resource.data.userId == request.auth.uid;
}

match /user_analytics/{eventId} {
  allow read: if request.auth != null;
  allow create: if request.auth != null
    && (request.resource.data.userId == request.auth.uid || request.resource.data.userId == null);
}

match /search_logs/{logId} {
  allow read: if request.auth != null;
  allow create: if true;
}
```

Storage:

```js
match /reviews/{poiId}/{userId}/{fileName} {
  allow read: if true;
  allow write: if request.auth != null && request.auth.uid == userId;
}
```
